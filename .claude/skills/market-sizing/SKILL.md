# 市場規模試算（TAM/SAM/SOM）

市場機会を定量的に評価し、事業計画やピッチに必要な市場規模を算出する。

## Purpose

新規事業や製品の市場機会を定量化し、投資判断や戦略立案の根拠を提供する。「この市場は追求する価値があるか？」を数字で答える。

## Key Concepts

### TAM/SAM/SOM

```
┌─────────────────────────────────────┐
│              TAM                     │ ← 全体市場
│  ┌─────────────────────────────┐    │
│  │           SAM               │    │ ← 獲得可能市場
│  │  ┌───────────────────┐     │    │
│  │  │       SOM         │     │    │ ← 実現可能市場
│  │  └───────────────────┘     │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

| 指標 | 説明 | 例 |
|------|------|-----|
| **TAM** (Total Addressable Market) | 製品カテゴリ全体の市場規模 | 全世界のプロジェクト管理ツール市場 |
| **SAM** (Serviceable Addressable Market) | 自社が提供可能な範囲の市場 | 日本の中小企業向けPM市場 |
| **SOM** (Serviceable Obtainable Market) | 現実的に獲得できる市場 | 今後3年で獲得見込みのシェア |

### 2つのアプローチ

| アプローチ | 方法 | 適した状況 |
|-----------|------|-----------|
| **トップダウン** | 全体市場→自社シェアで算出 | 既存市場、業界レポートあり |
| **ボトムアップ** | 顧客数×単価で積み上げ | 新規市場、独自データあり |

## Application

### Step 1: 対象市場の定義

以下を明確にする:

1. **製品カテゴリ**: 何の市場か？
2. **地域**: どの地域を対象とするか？
3. **顧客セグメント**: 誰が顧客か？
4. **時間軸**: いつ時点の市場か？

### Step 2: アプローチの選択

**トップダウンが適する場合**:
- 業界レポートが入手可能
- 既存市場に参入
- 投資家向けのピッチ

**ボトムアップが適する場合**:
- 新しい市場カテゴリ
- 独自の顧客データがある
- より精緻な予測が必要

### Step 3-A: トップダウン計算

```markdown
## トップダウン計算

### TAM（全体市場）
- 情報源: [業界レポート名、調査会社]
- 全体市場規模: [金額]
- 成長率: [年率%]

### SAM（獲得可能市場）
TAM × [地域フィルター] × [セグメントフィルター]

計算:
- TAM: [金額]
- 日本市場比率: [%]
- ターゲットセグメント比率: [%]
- **SAM = [金額]**

### SOM（実現可能市場）
SAM × [獲得可能シェア]

計算:
- SAM: [金額]
- 3年後目標シェア: [%]
- **SOM = [金額]**
```

### Step 3-B: ボトムアップ計算

```markdown
## ボトムアップ計算

### 顧客数の推計

| セグメント | 企業数 | 対象率 | 潜在顧客数 |
|-----------|--------|--------|-----------|
| 大企業 | [数] | [%] | [数] |
| 中小企業 | [数] | [%] | [数] |
| **合計** | | | **[数]** |

### 単価の設定

| プラン | 月額 | 年額 |
|--------|------|------|
| ベーシック | ¥[金額] | ¥[金額] |
| プロ | ¥[金額] | ¥[金額] |
| **平均** | ¥[金額] | ¥[金額] |

### TAM計算
潜在顧客数 × 年間平均単価 = TAM

計算:
- 潜在顧客数: [数]
- 年間平均単価: ¥[金額]
- **TAM = ¥[金額]**

### SAM/SOM計算
[地域・セグメントで絞り込み]
```

### Step 4: 検証と調整

- トップダウンとボトムアップの結果を比較
- 大きな乖離があれば仮定を見直し
- 感度分析（楽観・標準・悲観シナリオ）

### Step 5: 結果の整理

```markdown
## 市場規模サマリー

| 指標 | 金額 | 根拠 |
|------|------|------|
| TAM | ¥[金額] | [根拠] |
| SAM | ¥[金額] | [根拠] |
| SOM（3年） | ¥[金額] | [根拠] |

### 主要な仮定
1. [仮定1]
2. [仮定2]

### リスク要因
1. [リスク1]
2. [リスク2]
```

## Examples

### 例: SaaSプロジェクト管理ツール

```markdown
## 市場規模試算: プロジェクト管理SaaS

### 対象市場の定義
- 製品: クラウドベースのプロジェクト管理ツール
- 地域: 日本
- 顧客: 従業員50-500名の中小企業
- 時点: 2025年

---

## トップダウン計算

### TAM
- 情報源: Gartner, IDC
- グローバル市場: $7.5B（約1.1兆円）
- 成長率: 年10%

### SAM
- 日本市場比率: 8%
- 中小企業向け比率: 40%
- **SAM = 1.1兆円 × 8% × 40% = 352億円**

### SOM
- 競合: 10社以上
- 3年後目標シェア: 2%
- **SOM = 352億円 × 2% = 7億円**

---

## ボトムアップ計算

### 顧客数
- 日本の中小企業（50-500名）: 約5万社
- PMツール導入意向: 30%
- **潜在顧客: 1.5万社**

### 単価
- 平均月額: ¥50,000/社
- 年額: ¥600,000/社

### TAM
- 1.5万社 × ¥60万 = **900億円**

### SAM
- 実際にリーチ可能（マーケ予算制約）: 40%
- **SAM = 360億円**

### SOM
- 3年で獲得見込み: 200社
- **SOM = 200社 × ¥60万 = 1.2億円**

---

## クロスチェック

| 指標 | トップダウン | ボトムアップ | 採用値 |
|------|-------------|-------------|--------|
| TAM | 1,100億円 | 900億円 | 1,000億円 |
| SAM | 352億円 | 360億円 | 350億円 |
| SOM | 7億円 | 1.2億円 | 3億円（保守的） |

## 結論
- 市場は十分大きい（TAM 1,000億円）
- 現実的な3年目標は3億円（シェア0.9%）
- 年間ARR 1億円達成で次ラウンド調達可能
```

## Common Pitfalls

| 落とし穴 | 結果 | 対策 |
|---------|------|------|
| TAMを大きく見せすぎ | 信頼性低下 | SAM/SOMで現実的な絞り込み |
| 古いデータを使用 | 判断を誤る | 直近2年以内のデータを使用 |
| 仮定を明示しない | 検証不能 | すべての仮定を記録 |
| 一つのアプローチのみ | 偏った結果 | 両アプローチでクロスチェック |
| 競合を無視 | SOMが非現実的 | 競合シェアを考慮 |
