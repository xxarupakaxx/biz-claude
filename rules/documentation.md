# Documentation Rules

ドキュメント作成時に適用されるルール。

## 構成・構造

- 目的と対象読者を最初に明確化
- 論理的な構成（概要→詳細→まとめ）
- 適切な見出しレベルの使用（H1→H2→H3）
- 長文は段落・箇条書きで分割

## 読みやすさ

- 一文は短く（60文字以内を目安）
- 専門用語には説明を付ける
- 具体例を適宜挿入
- 視覚的要素（表、図、リスト）を活用

## 一貫性

- 用語の統一（同じ概念に異なる言葉を使わない）
- 文体の統一（です・ます調 or である調）
- フォーマットの統一（日付形式、数値形式等）

## 品質チェックリスト

作成後に以下を確認:
- [ ] 目的が明確か
- [ ] 対象読者に適切な内容か
- [ ] 論理的に構成されているか
- [ ] 誤字脱字がないか
- [ ] 情報が正確か
- [ ] 必要な情報が網羅されているか
