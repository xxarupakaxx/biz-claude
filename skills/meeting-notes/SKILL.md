# 議事録・MTGサマリー作成

会議メモを構造化された議事録に変換し、アクションアイテムを明確にする。

## Purpose

会議の内容を効率的に整理し、参加者が次のアクションを明確に理解できる形式にまとめる。

## Key Concepts

### 議事録の構成要素

1. **会議情報**: 日時、参加者、目的
2. **アジェンダ**: 議題一覧
3. **議論内容**: 各議題の詳細
4. **決定事項**: 合意された内容
5. **アクションアイテム**: 誰が・何を・いつまでに
6. **次回予定**: 次の会議の日程・議題

### アクションアイテムの形式

```
- [ ] [担当者]: [タスク内容] (期限: YYYY-MM-DD)
```

## Application

### Step 1: 会議メモの読み取り

ユーザーから提供された会議メモを読み取り:
- 参加者を特定
- 議題を抽出
- 議論の流れを把握

### Step 2: 構造化

以下のテンプレートに整理:

```markdown
# [会議名] 議事録

## 会議情報
- 日時: YYYY-MM-DD HH:MM - HH:MM
- 場所: [場所/オンライン]
- 参加者: A, B, C
- 記録者: [名前]

## アジェンダ
1. [議題1]
2. [議題2]
3. [議題3]

## 議論内容

### 1. [議題1]
**概要**: [議論の概要]

**主な意見**:
- Aさん: [意見]
- Bさん: [意見]

**結論**: [決定事項]

### 2. [議題2]
...

## 決定事項
- [決定1]
- [決定2]

## アクションアイテム
- [ ] A: [タスク1] (期限: YYYY-MM-DD)
- [ ] B: [タスク2] (期限: YYYY-MM-DD)

## 次回予定
- 日時: YYYY-MM-DD HH:MM
- 議題: [予定議題]
```

### Step 3: 確認

- アクションアイテムの担当者・期限が明確か確認
- 不明点があればAskUserQuestionで確認

## Examples

### 例: 週次ミーティングの議事録化

```
入力（生のメモ）:
---
3/8 週次MTG
田中、佐藤、山田
・プロジェクトAの進捗確認
  → 予定通り。来週リリース予定
  → 田中さんがテスト完了させる
・新機能Bの検討
  → 佐藤さんが要件まとめる
  → 次回MTGで決定
・その他
  → 来週は水曜10時に変更
---

出力:
# 週次ミーティング 議事録

## 会議情報
- 日時: 2026-03-08
- 参加者: 田中、佐藤、山田

## 議論内容

### 1. プロジェクトAの進捗確認
**概要**: 進捗状況の確認
**結論**: 予定通り進行中。来週リリース予定。

### 2. 新機能Bの検討
**概要**: 新機能の方向性検討
**結論**: 要件をまとめた上で次回決定。

## アクションアイテム
- [ ] 田中: テスト完了 (期限: 2026-03-14)
- [ ] 佐藤: 新機能Bの要件まとめ (期限: 2026-03-14)

## 次回予定
- 日時: 2026-03-12 (水) 10:00
```

## Common Pitfalls

| 落とし穴 | 結果 | 対策 |
|---------|------|------|
| アクションアイテムの担当者不明 | タスクが宙に浮く | 必ず担当者を明記 |
| 期限なし | フォローアップできない | デフォルトで次回MTGを期限に |
| 議論と決定の混同 | 何が決まったか不明確 | 「決定事項」セクションを分離 |
