# Jobs-to-be-Done（JTBD）分析

顧客が本当に達成しようとしていること（ジョブ）を理解し、製品の機会を特定する。

## Purpose

顧客の行動の背後にある動機を理解し、ソリューションに依存しない形でニーズを特定する。これにより、真に価値のある製品・機能を設計できる。

## Key Concepts

### 3つのジョブカテゴリ

| カテゴリ | 説明 | 例 |
|---------|------|-----|
| **機能的ジョブ** | タスクを完了させたい | 請求書を送る、ファイルを共有する |
| **社会的ジョブ** | 他者からどう見られたいか | プロフェッショナルに見える、信頼される |
| **感情的ジョブ** | どう感じたいか | 安心する、自信を持つ |

### ペインとゲイン

- **ペイン**: ジョブを達成する上での障害、不満、リスク
- **ゲイン**: ジョブ達成時に期待する利益、望む結果

### JTBDステートメント形式

```
[状況]において、[ペルソナ]は[ジョブ]を達成したい。
なぜなら[動機・理由]だからだ。
```

## Application

### Step 1: コンテキスト収集

以下を確認:
1. 誰が顧客か？（ペルソナ）
2. どのような状況で製品を使うか？
3. 現在どのように課題を解決しているか？

### Step 2: ジョブの特定

顧客インタビューや行動観察から:
- **機能的ジョブ**: 「何をしようとしているか？」
- **社会的ジョブ**: 「他者にどう思われたいか？」
- **感情的ジョブ**: 「どう感じたいか？」

### Step 3: ペインとゲインの洗い出し

各ジョブに対して:
- 現状の不満・障害（ペイン）
- 期待する結果・利益（ゲイン）

### Step 4: JTBDキャンバス作成

```markdown
## JTBDキャンバス

### ペルソナ
[ターゲット顧客の説明]

### 状況
[製品を使う場面・コンテキスト]

### ジョブ

#### 機能的ジョブ
1. [ジョブ1]
2. [ジョブ2]

#### 社会的ジョブ
1. [ジョブ1]

#### 感情的ジョブ
1. [ジョブ1]

### ペイン
- [ペイン1]: [深刻度: 高/中/低]
- [ペイン2]: [深刻度: 高/中/低]

### ゲイン
- [ゲイン1]: [重要度: 高/中/低]
- [ゲイン2]: [重要度: 高/中/低]

### 機会
[ペインを解消し、ゲインを提供する製品機会]
```

### Step 5: 知見保存

JTBDキャンバスをmemories/に保存し、製品開発に活用。

## Examples

### 例: フリーランス向け請求書ツール

```markdown
## JTBDキャンバス

### ペルソナ
フリーランスのデザイナー（独立1-3年目）

### 状況
月末にクライアントへ請求書を送る時

### ジョブ

#### 機能的ジョブ
1. 正確な請求書を素早く作成したい
2. 支払い状況を追跡したい
3. 未払いを催促したい

#### 社会的ジョブ
1. クライアントにプロフェッショナルに見られたい
2. 「ちゃんとした事業者」と思われたい

#### 感情的ジョブ
1. お金の心配から解放されたい
2. 請求業務のストレスを減らしたい

### ペイン
- 請求書作成に時間がかかる: [深刻度: 高]
- 未払いの追跡が面倒: [深刻度: 高]
- 自作請求書が素人っぽく見える: [深刻度: 中]

### ゲイン
- 請求書作成時間を80%削減: [重要度: 高]
- 自動リマインダーで催促不要: [重要度: 高]
- プロフェッショナルなテンプレート: [重要度: 中]

### 機会
「見た目がプロフェッショナルで、作成が簡単、自動追跡機能付きの請求書ツール」
```

## Common Pitfalls

| 落とし穴 | 結果 | 対策 |
|---------|------|------|
| 機能をジョブと混同 | 本質的ニーズを見逃す | 「なぜそれが必要？」を繰り返す |
| 機能的ジョブのみ分析 | 感情面を無視した製品になる | 3カテゴリ全てを検討 |
| 想像でJTBDを作成 | 顧客の実態と乖離 | 必ずインタビュー・観察を行う |
| ジョブが抽象的すぎ | 具体的な機能に落とせない | 状況と行動を具体的に記述 |
