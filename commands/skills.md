# biz-claude スキル一覧

ビジネスタスクを効率化するための専用スキル。

## スキルの種類

| 種類 | 説明 |
|------|------|
| **Workflow** | 複数フェーズの完全なプロセス |
| **Interactive** | 質問→推奨の対話形式 |
| **Component** | 単一成果物の作成 |

---

## 引き継ぎ・ドキュメント系

### handover-docs（引き継ぎ資料作成）
**Type**: Workflow

GitHubリポジトリやプロジェクト資料から引き継ぎ資料を作成。

**使い方の例**:
- 「このリポジトリの引き継ぎ資料を作って」
- 「プロジェクトAの引き継ぎドキュメントを作成して」

**フロー**:
1. deepwikiでリポジトリ構造を理解
2. 重要ファイルを読み取り
3. 構造化された引き継ぎ資料を生成

---

### meeting-notes（議事録作成）
**Type**: Component

会議メモを構造化された議事録に変換。

**使い方の例**:
- 「この会議メモを議事録にまとめて」
- 「MTGメモを整理してアクションアイテムを抽出して」

**出力**:
- 構造化された議事録
- アクションアイテム一覧
- 次回予定

---

### report-writing（レポート作成）
**Type**: Component

調査結果やデータを元にレポートを作成。

**使い方の例**:
- 「調査結果をレポートにまとめて」
- 「分析結果を経営層向けにレポート化して」

**出力形式**:
- エグゼクティブサマリー
- 詳細分析
- 推奨アクション

---

## リサーチ・企画系

### business-research（リサーチ・調査）
**Type**: Interactive

調査目的に応じた情報収集を実行。

**使い方の例**:
- 「競合Aについて調べて」
- 「〇〇市場のトレンドを調査して」
- 「認証ライブラリを比較検討して」

**ツール**:
- WebSearch: 最新情報、市場情報
- deepwiki: リポジトリの設計理解
- Context7: ライブラリドキュメント

---

### brainstorming（アイデア探索）
**Type**: Interactive

アイデアを対話を通じて設計・仕様に昇華。

**使い方の例**:
- 「新機能のアイデアを整理したい」
- 「サービス改善の方向性を検討したい」
- 「このコンセプトを具体化したい」

**プロセス**:
1. 一度に1つの質問でアイデアを理解
2. 2-3のアプローチを提案
3. 段階的に設計を詳細化

---

### prd-writing（PRD作成）
**Type**: Workflow

製品要件定義書（PRD）を作成。

**使い方の例**:
- 「新機能のPRDを作成したい」
- 「この機能要件をPRDにまとめて」

**出力内容**:
- ユーザーストーリー
- 機能要件・非機能要件
- 受入基準（Gherkin形式）
- マイルストーン

---

## 知見管理系

### knowledge-capture（知見キャプチャ）
**Type**: Component

作業で得た知見を構造化して保存。

**使い方の例**:
- 「今回の調査結果を知見として保存して」
- 「この解決策を記録しておいて」

**保存先**:
- `memories/`: 検索用インデックス
- `solutions/`: 詳細な知見

---

## スキルの呼び出し方

### 方法1: 自然な依頼

```
「引き継ぎ資料を作って」
→ handover-docs スキルが起動
```

### 方法2: スキル名を指定

```
「business-researchスキルで競合を調べて」
→ business-research スキルが起動
```

---

他のコマンド:
- `/guide`: ガイドツアー
- `/workflow`: 作業フローの詳細
- `/lfg <タスク>`: 自律ワークフローで実行
