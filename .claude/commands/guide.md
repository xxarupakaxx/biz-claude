# biz-claude ガイドツアー

biz-claudeへようこそ！このガイドでは、ビジネス業務を効率化するためのClaude Code設定の使い方を説明します。

## 概要

biz-claudeは、以下のようなビジネスタスクを支援するClaude Code設定です:

- 引き継ぎ資料の作成
- リサーチ・調査
- 会議メモの構造化
- レポート作成
- PRD（製品要件定義書）作成
- アイデア探索・設計
- **PM業務**: ユーザーストーリー、優先順位付け、市場規模試算、JTBD分析

## 基本的な使い方

### 1. コマンドで機能を呼び出す

| コマンド | 説明 |
|---------|------|
| `/guide` | このガイドを表示 |
| `/workflow` | 作業フローの詳細を確認 |
| `/skills` | 利用可能なスキル一覧を表示 |
| `/lfg <タスク>` | 自律ワークフローでタスクを実行 |

### 2. スキルを活用する

特定のタスクには専用スキルを使用:

**ドキュメント・レポート系**
| スキル | 用途 | 使い方の例 |
|-------|------|-----------|
| `handover-docs` | 引き継ぎ資料作成 | 「このリポジトリの引き継ぎ資料を作って」 |
| `meeting-notes` | 議事録作成 | 「この会議メモを整理して」 |
| `report-writing` | レポート作成 | 「調査結果をレポートにまとめて」 |

**リサーチ・企画系**
| スキル | 用途 | 使い方の例 |
|-------|------|-----------|
| `business-research` | リサーチ・調査 | 「競合Aについて調べて」 |
| `brainstorming` | アイデア探索 | 「新しいサービスのアイデアを整理したい」 |

**PM（プロダクトマネジメント）系**
| スキル | 用途 | 使い方の例 |
|-------|------|-----------|
| `prd-writing` | PRD作成 | 「新機能のPRDを作成したい」 |
| `jobs-to-be-done` | 顧客ニーズ分析 | 「顧客のジョブを整理して」 |
| `user-story` | ユーザーストーリー | 「この機能のストーリーを書いて」 |
| `user-story-mapping` | ストーリーマップ | 「バックログを整理してMVPを決めたい」 |
| `prioritization-advisor` | 優先順位付け | 「何から着手すべき？」 |
| `market-sizing` | 市場規模試算 | 「この市場のTAMを計算して」 |

**知見管理・改善系**
| スキル | 用途 | 使い方の例 |
|-------|------|-----------|
| `knowledge-capture` | 知見保存 | 「この解決策を記録して」 |
| `agent-memory` | メモリ操作 | 「これ覚えて」「〇〇について思い出して」 |
| `cleanup-knowledge` | 知見整理 | 「溜まった知見を整理して」 |
| `update-inst` | 設定改善 | 「さっきの間違いを防ぐように設定を更新して」 |
| `ship-learn-next` | 学習→実行計画 | 「この記事の内容をアクションプランにして」 |
| `kaizen` | 継続的改善 | 「この業務フローを改善したい」 |

### 3. 知見を蓄積する

作業で得た知見は自動的に保存され、次回以降のタスクで活用できます:

- **memories/**: 検索用インデックス
- **solutions/**: 詳細な知見・解決策

## 作業フロー

biz-claudeは以下の5つのPhaseでタスクを進めます:

```
Phase 0: 準備
   ↓
Phase 1: 調査
   ↓
Phase 2: 作成
   ↓
Phase 3: レビュー
   ↓
Phase 4: 完了
```

詳細は `/workflow` で確認できます。

## よくあるユースケース

### ユースケース1: 引き継ぎ資料を作成したい

```
1. リポジトリURLを教えてください
2. 「引き継ぎ資料を作成して」と依頼
3. deepwikiでリポジトリを分析し、構造化された資料を作成
```

### ユースケース2: 競合について調べたい

```
1. 「競合Aについて調べて」と依頼
2. 調査の観点（機能比較、価格比較等）を確認
3. WebSearchで情報収集し、レポートを作成
```

### ユースケース3: 会議メモを整理したい

```
1. 会議メモを貼り付け
2. 「議事録にまとめて」と依頼
3. 構造化された議事録とアクションアイテムを出力
```
