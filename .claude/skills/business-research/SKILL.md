# ビジネスリサーチ

調査目的に応じた情報収集を行い、構造化された調査結果を作成する。

## Purpose

ビジネス判断に必要な情報を効率的に収集・整理し、意思決定をサポートする。

## Key Concepts

### 調査の種類

| 種類 | 目的 | 主なツール |
|------|------|-----------|
| 市場調査 | 市場規模・トレンド把握 | WebSearch |
| 競合調査 | 競合の動向・戦略分析 | WebSearch |
| 技術調査 | 技術選定・実現可能性 | deepwiki, Context7 |
| ユーザー調査 | ニーズ・課題の把握 | WebSearch |

### 情報源の使い分け

| ツール | 得意な情報 |
|--------|-----------|
| **WebSearch** | 最新ニュース、市場情報、一般的な情報 |
| **deepwiki** | GitHubリポジトリの設計・実装詳細 |
| **Context7** | ライブラリ/フレームワークのドキュメント |

## Application

### Step 1: 目的の明確化（質問）

以下を確認:
1. 調査の目的は何か？（例: 新機能の技術選定、競合分析）
2. どのような情報が必要か？
3. アウトプットの形式は？（レポート、比較表、サマリー）
4. 期限・スコープの制約は？

### Step 2: 調査計画

目的に応じた調査計画を提案:
- 使用する情報源
- 調査の順序
- 予想されるアウトプット

### Step 3: 情報収集

計画に沿って情報収集:
- 各情報源から関連情報を取得
- 重要な発見を記録
- 不明点があれば追加調査

### Step 4: 構造化

収集した情報を構造化（**10年後に読み返しても調査の価値と文脈が分かるように**）:

```markdown
# [調査テーマ] 調査レポート

> 作成日: YYYY-MM-DD ／ 調査者: [名前（役割）]
> 本レポートはYYYY年MM月時点の情報に基づいています。

## 調査概要
- **目的**: [なぜこの調査が必要になったか。きっかけ・背景]
- **調査期間**: YYYY-MM-DD 〜 YYYY-MM-DD
- **スコープ**: [何を対象にし、何を対象外としたか]
- **調査手法**: [どのように情報を収集・分析したか]
- **制約・限界**: [入手できなかった情報、調査の限界]

## 主な発見

### 発見1
[詳細]
**根拠**: [情報源とデータ]
**信頼度**: [高/中/低。公式発表か推測か]

### 発見2
[詳細]
**根拠**: [同上]

## 分析・考察
[発見からの示唆。なぜそう解釈できるかの論理展開]

## 推奨アクション
| アクション | 理由 | リスク・注意点 |
|-----------|------|--------------|
| [アクション1] | [なぜ推奨か] | [実行時の留意点] |

## 用語集（該当がある場合）
| 用語 | 説明 |
|------|------|
| [用語] | [定義] |

## 参考情報
| 情報源 | URL | 参照日 | 信頼度 |
|--------|-----|--------|--------|
| [名称] | [URL] | YYYY-MM-DD | [高/中/低] |
```

### Step 5: 知見保存

再利用価値のある発見をmemories/solutions/に保存。

## Examples

### 例1: 競合分析

```
ユーザー: 競合Aについて調べて

質問:
- どのような観点で調査しますか？
  - 機能比較
  - 価格比較
  - 市場ポジショニング

実行:
1. WebSearchで競合Aの最新情報を収集
2. 公式サイト・プレスリリースを確認
3. 比較表を作成
4. 考察・推奨アクションを追加
```

### 例2: 技術選定調査

```
ユーザー: 認証ライブラリを調べて

質問:
- 対象の技術スタックは？（例: Next.js, React）
- 重視する観点は？（セキュリティ、使いやすさ、コスト）

実行:
1. Context7で主要ライブラリのドキュメント確認
2. deepwikiで実装例を調査
3. 比較表を作成
4. 推奨ライブラリを提案
```

## Common Pitfalls

| 落とし穴 | 結果 | 対策 |
|---------|------|------|
| 目的の曖昧さ | 的外れな調査結果 | 最初に目的を明確化 |
| 単一ソース | 偏った情報 | 複数ソースを組み合わせ |
| 情報の鮮度無視 | 古い情報に基づく判断 | 日付を確認、最新情報を優先 |
| 調査手法の省略 | 再現・検証ができない | どう調べたかを記録 |
| 情報源の信頼度不明 | 根拠の強度が判断できない | 各発見に情報源と信頼度を付記 |
| 調査の前提・制約の省略 | 結論だけ独り歩きする | スコープ外と限界を正直に記載 |
