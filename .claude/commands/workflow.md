# biz-claude 作業フロー

ビジネスタスクを効率的かつ品質高く進めるための5つのPhase。

## フロー概要

```
Phase 0: 準備
   │
   ├── メモリディレクトリ作成
   ├── 05_log.md初期化
   └── 過去知見を検索
   │
   ▼
Phase 1: 調査
   │
   ├── 過去知見の参照（memories/, solutions/）
   ├── 外部情報収集（deepwiki, WebSearch, Context7）
   └── 調査結果を記録
   │
   ▼
Phase 2: 作成
   │
   ├── 構成の検討
   ├── ドラフト作成
   └── 必要に応じてユーザー確認
   │
   ▼
Phase 3: レビュー
   │
   ├── セルフチェック
   ├── サブエージェントレビュー（推奨）
   └── 修正・改善
   │
   ▼
Phase 4: 完了
   │
   ├── 完了報告
   └── 知見を保存（memories/, solutions/）
```

## 各Phaseの詳細

### Phase 0: 準備

タスク開始時の準備:

1. **メモリディレクトリ作成**: `.local/memory/YYMMDD_<task>/`
2. **作業ログ初期化**: 05_log.md作成、最初の指示を記録
3. **過去知見検索**: memories/とsolutions/から関連情報を検索

### Phase 1: 調査

情報収集フェーズ:

| ツール | 用途 |
|--------|------|
| **過去知見** | memories/, solutions/から類似タスクを検索 |
| **deepwiki** | GitHubリポジトリの構造・設計理解 |
| **WebSearch** | 最新情報、一般的な情報収集 |
| **Context7** | ライブラリのドキュメント参照 |

**重要**: 外部情報参照は必須（最低1回）

### Phase 2: 作成

成果物を作成:

1. **構成検討**: 目的・対象読者を明確化、セクション構成を決定
2. **ドラフト作成**: 構成に沿って作成、05_log.mdに状況を記録
3. **ユーザー確認**: 大規模な場合は方向性を確認

### Phase 3: レビュー

品質確認:

| チェック項目 | 内容 |
|------------|------|
| 情報の正確性 | ソースと照合 |
| 論理的整合性 | 矛盾がないか |
| 読みやすさ | 構成、文章 |
| 抜け漏れ | 必要な情報が揃っているか |

**推奨**: reviewerエージェントで第三者視点のレビュー

### Phase 4: 完了

タスク終了処理:

1. **完了報告**:
   - 作成した成果物の概要
   - 自律決定した事項
   - 残存する課題

2. **知見保存**:
   - 価値ある発見をmemories/に保存
   - 詳細な解決策をsolutions/に保存

## 知見の複利ループ

```
タスク実行 → 知見保存（memories/, solutions/）
     ↑                    │
     └────────────────────┘
         次のタスクで活用
```

過去に蓄積した知見は、将来のタスクで自動的に参照されます。

## よくあるタスクパターン

### パターン1: 引き継ぎ資料

```
Phase 0 → Phase 1（deepwikiで調査）→ Phase 2（資料作成）
       → Phase 3（レビュー）→ Phase 4（知見保存）
```

### パターン2: リサーチ

```
Phase 0 → Phase 1（WebSearch/deepwikiで情報収集）
       → Phase 2（レポート作成）→ Phase 3（確認）
       → Phase 4（知見保存）
```

### パターン3: 会議メモ整理

```
Phase 0 → Phase 1（メモ読み取り）→ Phase 2（議事録作成）
       → Phase 3（AI確認・チェック）→ Phase 4（完了）
```
